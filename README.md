# VQMEA
🧠 VQMEA: Generative Multi-Modal Entity Alignment with Open Knowledge-Enhanced Quantization
📝 Abstract

Multi-Modal Entity Alignment (MMEA) aims to identify equivalent entities across heterogeneous multi-modal knowledge graphs (MMKGs). Existing embedding-based approaches have achieved notable progress through sophisticated cross-modal fusion strategies. However, they suffer from two key limitations: (1) insufficient prior knowledge to handle highly heterogeneous entities, and (2) limited reasoning and generalization capability due to reliance on static embedding spaces.

To address these challenges, we propose VQMEA, a novel framework that enables generative multi-modal entity alignment via vector quantization and large language models (LLMs). Specifically, VQMEA introduces an LLM-enhanced multi-modal encoding module, where open-world knowledge is injected as semantic anchors to guide representation learning. Then, a residual vector quantization module converts continuous embeddings into hierarchical discrete token sequences, bridging the gap between embedding spaces and LLM token spaces.

Furthermore, we incorporate multi-modal contrastive learning objectives and semantic anchor constraints to enhance alignment robustness. Finally, we construct instruction-based data to fine-tune LLMs, enabling entity alignment to be performed in a generative manner instead of similarity matching.

Extensive experiments on five benchmark datasets demonstrate that VQMEA achieves state-of-the-art performance and provides a new paradigm for integrating LLMs with MMEA tasks .

🚀 How to Run

VQMEA consists of three main stages:

1️⃣ Multi-Modal Encoding with LLM Enhancement

Encode entities from multiple modalities (graph, image, relation, attribute), and inject open-world knowledge generated by LLMs as semantic anchors.

python MMEA_model/run.py \
--dataset FB15K-DB15K \
--use_llm_anchor \
--use_gating_fusion

This stage produces fused multi-modal entity embeddings.

2️⃣ Residual Quantization

Convert continuous embeddings into discrete token sequences via multi-stage residual vector quantization.

python quantitative_code/get_code.py \
--codebook_size 1000 \
--num_stages 16

This step generates hierarchical code sequences that represent entity semantics.

3️⃣ LLM Fine-tuning for Generative Alignment
Stage 1: Token Integration

Add quantized codes as new tokens into the LLM vocabulary.

python token_expand.py
Stage 2: Instruction Fine-tuning

Train the LLM to perform entity alignment via instruction-based generation.

python finetune_lora.py \
--model llama2 \
--task entity_alignment
🧩 Key Features
🔹 Open Knowledge-Enhanced Encoding
Inject LLM-generated descriptions as semantic anchors to improve representation quality
🔹 Residual Vector Quantization
Transform continuous multi-modal embeddings into hierarchical discrete token sequences
🔹 Generative Entity Alignment
Reformulate MMEA as a sequence generation problem instead of similarity matching
🔹 Semantic Anchor Constraint
Preserve open-world knowledge and improve robustness under high heterogeneity
🔹 Multi-Modal Contrastive Learning
Strengthen alignment capability across modalities and knowledge graphs
🔹 LLM Instruction Tuning
Enable reasoning-based alignment via prompt-driven generation

📊 VQMEA achieves state-of-the-art performance across five benchmark datasets, significantly outperforming existing MMEA methods, especially on highly heterogeneous cross-KG datasets .
